# Corner Grocer Item-Tracking Program

## Project Summary

The project I selected for my GitHub portfolio is the Corner Grocer item-tracking program. This program was created to help a grocery store analyze a text file of purchased items and determine how often each item appears. The program gives the user a menu with options to search for a specific item, display the frequency of all items, and show a histogram of item frequencies. The problem it solves is helping the store better understand purchasing patterns so they can make smarter inventory decisions.

## What I Did Well

One thing I did particularly well was creating a clear menu-driven program that is easy for the user to understand. The program gives simple options and produces readable output. I also organized the code so that the main parts of the program, such as reading the file, counting items, and displaying results, were easier to follow. This made the program more readable and easier to test.

## Where I Could Enhance the Code

One area where I could improve the code is input validation. I could add stronger checks to make sure the user only enters valid menu choices. I could also improve how item searches are handled by making the search ignore capitalization differences, so searching for “apple” and “Apple” would give the same result. These improvements would make the program more reliable, secure, and user-friendly. I could also reduce repeated code by using more functions, which would make the program easier to update later.

## Challenges and How I Overcame Them

The most challenging part of this project was working with file input and output. I had to make sure the program correctly read the input file, counted the frequency of each item, and created the frequency data file. I overcame this by breaking the project into smaller steps and testing each part as I built it. I used course resources, Visual Studio debugging tools, and examples from class to help me solve problems.

## Transferable Skills

The skills from this project that will transfer to other projects include working with files, using loops and conditionals, creating functions, building menu-based programs, and testing code for errors. These skills are useful because many real-world programs need to read data, process it, and display useful results to the user.

## Maintainability, Readability, and Adaptability

I made this program maintainable, readable, and adaptable by using meaningful variable names, organizing the code into clear sections, and adding comments to explain important parts of the program. The menu structure also makes the program adaptable because more options could be added in the future without rewriting the whole program. Overall, this project helped me practice writing organized and functional C++ code that solves a real-world problem.
